# الأمان: تسجيل الخروج وإلغاء الجلسات

## المشكلة المبلّغة
"لو فتحت رابطاً عميقاً مثل `/instructor/sessions` يدخل بدون تسجيل دخول."

## التشخيص
كود الحماية في المستودع **سليم** أصلاً:
- الواجهة: `RequireRole` يحوّل غير المسجّل إلى `/login` (`src/auth/guards.tsx`).
- الخادم: `DEFAULT_PERMISSION_CLASSES = IsAuthenticated` — كل الـ endpoints تتطلّب
  مصادقة افتراضياً.

فمتصفح **جديد بلا توكن** يُحوَّل حتماً إلى صفحة الدخول. سبب ظهور "يدخل بدون تسجيل" أحد
أمرين:
1. المتصفح الذي جُرِّب كان يملك **جلسة صالحة محفوظة** (التوكن في `localStorage`، صالح 7
   أيام) — فالرابط يفتح مباشرةً لأنك مسجّل أصلاً؛ الرابط نفسه لا يحمل بيانات الدخول.
2. أو أن الموقع المنشور (`oneclup.com`) يشغّل **نسخة أقدم** من الكود المصحّح.

الضعف الحقيقي الذي عالجناه: **لم يكن هناك تسجيل خروج فعلي من الخادم ولا إمكانية إلغاء
التوكن** — فالجلسة تبقى صالحة 7 أيام ولا يمكن إبطالها حتى بعد "الخروج".

## ما تم إصلاحه
- تفعيل تطبيق `rest_framework_simplejwt.token_blacklist` (إلغاء التوكن على الخادم).
- `ROTATE_REFRESH_TOKENS = True` و `BLACKLIST_AFTER_ROTATION = True`: كل تحديث
  يُصدر refresh token جديداً ويُدرِج القديم في القائمة السوداء — فالتوكن المسروق يُستخدم
  مرّة واحدة كحدّ أقصى.
- endpoint حقيقي لتسجيل الخروج: `POST /auth/logout/` (يأخذ `refresh` ويُبطله).
- الواجهة: `authApi.logout()` صار ينادي `/auth/logout/` لإبطال التوكن على الخادم قبل
  مسحه محلياً.
- أعمار التوكن صارت قابلة للضبط بيئياً: `JWT_ACCESS_MINUTES` (افتراضي 30) و
  `JWT_REFRESH_DAYS` (افتراضي 7).

## خطوات النشر (مهم)
```bash
python manage.py migrate           # ينشئ جداول token_blacklist
# اختياري: تقصير عمر الجلسة على الأجهزة المشتركة
export JWT_REFRESH_DAYS=1
# اختياري دوري: تنظيف التوكنات المنتهية
python manage.py flushexpiredtokens
```

## كيف تتحقق
افتح `https://oneclup.com/instructor/sessions` في نافذة **خفية (Incognito)** بلا جلسة:
يجب أن يحوّلك إلى `/login`. إن لم يحدث ذلك على الموقع المنشور، فالنسخة المنشورة أقدم من
هذا الإصلاح — أعد نشر آخر `main`.

## ملاحظة عن الـ access token
إبطال الـ refresh يمنع تمديد الجلسة، لكن الـ access token الحالي يظل صالحاً حتى انتهائه
(≤ 30 دقيقة) بحكم طبيعة JWT. لقتل فوري تام يمكن لاحقاً تقصير عمر الـ access أو إضافة
قائمة سوداء له.
